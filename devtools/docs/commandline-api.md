</header>    <main id="gc-pagecontent" role="main"><article class="article-content">  <div itemprop="articleBody"></p><h1 id="command-line-api-reference">Command Line API Reference</h1><p>The Command Line API is a collection of functions for performing common tasks with the Chrome Developer Tools. These include convenience functions for selecting and inspecting elements in the DOM, stopping and starting the profiler, and monitoring DOM events. This API complements the <a href="console-api.md">Console API</a>, the Command Line API is only available from within the console itself.</p><h2 id="_">$_</h2><p>Returns the value of the most recently evaluated expression. In the following example, a simple expression is evaluated. The <code>$_</code> property is then evaluated, which contains the same value:</p><p><img src="commandline-api-files/last_expression.png" class="screenshot"/></p><p>In the next example, the evaluated expression is a call to the <a href="commandline-api.md#selector_1"><code>$$()</code></a> method, which returns an array of elements that match the CSS selector. It then evaluates <code>$_.length</code> to get the length of the array (17), which becomes the most recently evaluated expression.</p><p><img src="commandline-api-files/last_expression_2.png" class="screenshot"/></p><h2 id="0-4">$0 - $4</h2><p>Dev Tools remembers the last five DOM elements (or JavaScript heap objects) that you've selected in the  tab (or Profiles panel). It makes those objects available as <strong><code>$0</code></strong>, <strong><code>$1</code></strong>, <strong><code>$2</code></strong>, <strong><code>$3</code></strong>, and <strong><code>$4</code></strong>. <strong><code>$0</code></strong> returns the most recently selected element or JavaScript object, <strong><code>$1</code></strong> returns the second most recently selected one, and so on.</p><p>In the following example, an element with the ID <code>gc-sidebar</code> is selected in the Elements tab. In the Console window <code>$0</code> has been evaluated, which displays the same element.</p><p><img alt="Example of $0" src="commandline-api-files/$0.png" /></p><p>The image below shows the <code>gc-content</code> element selected in the same page. The <strong><code>$0</code></strong> now refers to newly selected element, while <strong><code>$1</code></strong> now returns the previously selected one (<code>gc-sidebar</code>).</p><p><img alt="image alt text" src="commandline-api-files/$1.png" /></p><h2 id="selector">$(selector)</h2><p>Returns reference to the first DOM element with the specified CSS selector.This function is an alias for <a href="http://docs.webplatform.org/wiki/css/selectors_api/querySelector"><code>document.querySelector()</code></a> function.</p><p>The following example saves a reference to the first <code>&lt;img&gt;</code> element in the document and calls displays its <code>src</code> property:</p><pre><code>$('img').src;</code></pre><p><img alt="image alt text" src="commandline-api-files/$img_src.png" /></p><h2 id="selector_1">$$(selector)</h2><p>Returns an array of elements that match the given CSS selector. This command is equivalent to calling <a href="http://docs.webplatform.org/wiki/css/selectors_api/querySelectorAll"><code>document.querySelectorAll()</code></a>.</p><p>The following example uses <code>$$()</code> to create an array of all <code>&lt;img&gt;</code> elements in the current document and displays the value of each element's <code>src</code> property.</p><pre><code>var images = $$('img');for (each in images) {    images[each].src;}</code></pre><p><img alt="Example of using $$() to select all images in the document and display their sources." src="commandline-api-files/$$img_src.png" /></p><p>Note: Press Shift+Enter in the console to start a new line without executing the script.</p><h2 id="xpath">$x(path)</h2><p>Returns an array of DOM elements that match the given XPath expression. For example, the following returns all the <code>&lt;p&gt;</code> elements that contain <code>&lt;a&gt;</code> elements:</p><pre><code>$x("//p[a]")</code></pre><p><img alt="Example of using an Xpath selector" src="commandline-api-files/$xpath.png" /></p><h2 id="clear">clear()</h2><p>Clears the console of its history.</p><pre><code>clear();</code></pre><p>Also see <a href="console.md#clearing-the-console-history">Clearing the console</a>.</p><h2 id="copyobject">copy(object)</h2><p>Copies a string representation of the specified object to the clipboard.</p><pre><code>copy($0);</code></pre><h2 id="dirobject">dir(object)</h2><p>Displays an object-style listing of all the properties of the specified object. This method is an alias for the Console API's <a href="console-api.md#consoledirobject"><code>console.dir()</code></a> method.</p><p>The following example shows the difference between evaluating <code>document.body</code> directly in the command line, and using <code>dir()</code> to display the same element.</p><pre><code>document.body;dir(document.body);</code></pre><p><img alt="Logging document.body with and without dir() function" src="commandline-api-files/document.body.png" /></p><p>For more information, see the <a href="console-api.md#consoledirobject"><code>console.dir()</code></a> entry in the Console API.</p><h2 id="dirxmlobject">dirxml(object)</h2><p>Prints an XML representation of the specified object, as seen in the Elements tab. This method is equivalent to the <a href="console-api.md#consoledirxmlobject"><code>console.dirxml()</code></a> method.</p><h2 id="inspectobject">inspect(object)</h2><p>Opens and selects the specified element or object in the appropriate panel: either the Elements panel for DOM elements and the Profiles panel for JavaScript heap objects.</p><p>The following example opens the first child element of <code>document.body</code> in the Elements panel:</p><pre><code>inspect(document.body.firstChild);</code></pre><p><img alt="Inspecting an element with inspect()" src="commandline-api-files/inspect.png" /></p><h2 id="geteventlistenersobject">getEventListeners(object)</h2><p>Returns the event listeners registered on the specified object. The return value is an object that contains an array for each registered event type ("click" or "keydown", for example). The members of each array are objects that describe the listener registered for each type. For example, the following lists all the event listeners registered on the <code>document</code> object.</p><pre><code>getEventListeners(document);</code></pre><p><img alt="Output of using getEventListeners()" src="commandline-api-files/geteventlisteners_short.png" /></p><p>If more than one listener is registered on the specified object, then the array contains a member for each listener. For instance, in the following example there are two event listeners registered on the <code>#scrollingList</code> element for the "mousedown" event:</p><p><img alt="" src="commandline-api-files/geteventlisteners_multiple.png" /></p><p>You can further expand each of these objects to explore their properties:</p><p><img alt="Expanded view of listener object" src="commandline-api-files/geteventlisteners_expanded.png" /></p><h2 id="keysobject">keys(object)</h2><p>Returns an array containing the names of the properties belonging to the specified object. To get the associated values of the same properties, use <a href="commandline-api.md#valuesobject">values()</a>.</p><p>For example, suppose your application defined the following object:</p><pre><code>var player1 = {    "name": "Ted",    "level": 42}</code></pre><p>Assuming <code>player1</code> was defined in the global namespace (for simplicity), typing <code>keys(player1)</code> and <code>values(player1)</code> in the Console would result in the following:</p><p><img src="commandline-api-files/keys-values2.png" style="max-width:100%" alt="Example of keys() and values() methods"></p><h2 id="monitoreventsobject-events">monitorEvents(object[, events])</h2><p>When one of the specified events occurs on the specified object, the Event object is logged to the console. You can specify a single event to monitor, an array of events, or one of the generic events "types" that are mapped to a predefined collection of events. See examples below.</p><p>The following monitors all <code>resize</code> events on the <code>window</code> object.</p><pre><code>monitorEvents(window, "resize");</code></pre><p><img alt="Monitoring window resize events" src="commandline-api-files/monitor-resize.png" /></p><p>The following defines an array to monitor both "resize" and "scroll" events on the <code>window</code> object:</p><pre><code>monitorEvents(window, ["resize", "scroll"])</code></pre><p>You can also specify one of the available event "types", strings that map to predefined sets of events. The table below lists the available event types and their associated event mappings:</p><table><thead><tr><th>Event type</th><th>Corresponding mapped events</th></tr></thead><tbody><tr><td><strong>mouse</strong></td><td>"<code>mousedown</code>", "<code>mouseup</code>", "<code>click</code>", "<code>dblclick</code>", "<code>mousemove</code>", "<code>mouseover</code>", "<code>mouseout</code>", "<code>mousewheel</code>"</td></tr><tr><td><strong>key</strong></td><td>"<code>keydown</code>", "<code>keyup</code>", "<code>keypress</code>", "<code>textInput</code>"</td></tr><tr><td><strong>touch</strong></td><td>"<code>touchstart</code>", "<code>touchmove</code>", "<code>touchend</code>", "<code>touchcancel</code>"</td></tr><tr><td><strong>control</strong></td><td>"<code>resize</code>", "<code>scroll</code>", "<code>zoom</code>", "<code>focus</code>", "<code>blur</code>", "<code>select</code>", "<code>change</code>", "<code>submit</code>", "<code>reset</code>"</td></tr></tbody></table><p>For example, the following uses the "key" event type all corresponding key events on an input text field ("<code>#msg</code>").</p><pre><code>monitorEvents($("#msg"), "key");</code></pre><p>Below is sample output after typing two characters in the text field:</p><p><img alt="Monitoring key events" src="commandline-api-files/monitor-key-events.png" /></p><p><img src="commandline-api-files/monitor-key-events.png" class="screenshot" alt=""></p><h2 id="profilename">profile([name])</h2><p>Starts a JavaScript CPU profiling session with an optional name. To complete the profile call <a href="commandline-api.md#profileendname"><code>profileEnd()</code></a>.</p><p>To start profiling:</p><pre><code>profile("My profile")</code></pre><p>To stop profiling and display the results in the Profiles panel:</p><pre><code>profileEnd("My profile")</code></pre><p>For more examples, see <a href="console.md#controlling-the-cpu-profiler">Controlling the CPU profiler</a>.</p><h2 id="profileendname">profileEnd([name])</h2><p>Stops the current profiling session started with the <a href="commandline-api.md#profilename">profile()</a> method and displays the results in the Profiles panel.</p><h2 id="unmonitoreventsobject-events">unmonitorEvents(object[, events])</h2><p>Stops monitoring events for the specified object and events. For example, the following stops all event monitoring on the <code>window</code> object:</p><pre><code>unmonitorEvents(window);</code></pre><p>You can also selectively stop monitoring specific events on an object. For example, following code starts monitoring all mouse events on the currently selected element, and then stops monitoring "mousemove" events (perhaps to reduce noise in the console output).</p><pre><code>monitorEvents($0, "mouse");unmonitorEvents($0, "mousemove");</code></pre><p>Also see <a href="console.md#monitoring-events">Monitoring events</a>.</p><h2 id="valuesobject">values(object)</h2><p>Returns an array containing the values of all properties belonging to the specified object.</p><pre><code>values(object);</code></pre><p>        <footer id="cc-info">      <button id="send-feedback" class="google-button" data-feedback>Send Feedback</button>      <!--p class="last-updated">Last updated August 2, 2013.</p-->      <p class="cc-logo"><a href="http://creativecommons.org/licenses/by/3.0/"><img src="../../static/images/cc_by.png" alt="Licensed Creative Commons by Attribution"></a></p>    </footer>    <nav class="inline-toc no-permalink">      <section class="related">                  <h3 title="">Reference</h3>      <ol class="toc">                        <li>                          <a href="console-api.md">Console API Reference</a>                        </li>                        <li>                          <a href="commandline-api.md" class="active">Command Line API Reference</a>                        </li>                        <li>                          <a href="integrating.md">DevTools Extensions API</a>                        </li>                        <li>                          <a href="shortcuts.html">Keyboard Shortcuts</a>                        </li>                        <li>                          <a href="settings.html">Settings</a>                        </li>                        <li>                          <a href="debugger-protocol.html">Remote Debugging Protocol</a>                        </li>      </ol>      </section>      <section id="toc">
