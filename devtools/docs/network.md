</header>    <main id="gc-pagecontent" role="main"><article class="article-content">  <div itemprop="articleBody"></p><h1 id="evaluating-network-performance">Evaluating network performance</h1><p>The Network panel records information about each network operation in your application, including detailed timing data, HTTP request and response headers, cookies, WebSocket data, and more. The Network panel helps you answer questions about the network performance of your web application, such as:</p><ul><li>Which resource had the slowest time to first byte?</li><li>Which resources took the longest time to load (duration)?</li><li>Who initiated a particular network request?</li><li>How much time was spent in the various network phases for a particular resource?</li></ul><h2 id="about-the-resource-timing-api">About the Resource Timing API</h2><p>The Network panel uses the <a href="http://www.w3.org/TR/resource-timing">Resource Timing API</a>, a JavaScript API that provides detailed network timing data for each loaded resource. For example, the API can tell you precisely when the HTTP request for an image started, and when the image's final byte was received. The following illustration shows the network timing data points that the Resource Timing API provides.</p><p><img src="network-files/resource-timing-overview.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>The API is available to any web page, not just DevTools. In Chrome, it's exposed as methods on the global <code>window.performance</code> object. The <code>performance.getEntries()</code> method returns an array of "resource timing objects", one for each requested resource on the page.</p><p>Try this: open the JavaScript console on the current page, enter the following at the prompt, and hit Return:</p><pre><code>window.performance.getEntries()[0]</code></pre><p>This evaluates the first element in the array of resource timing objects and displays its properties in the console, as shown below.</p><p><img src="network-files/getentries.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>Each timestamp is in microseconds, following the <a href="http://www.w3.org/TR/hr-time/#sec-high-resolution-time">High ResolutionTime</a> specification. This API is <a href="http://updates.html5rocks.com/2012/08/When-milliseconds-are-not-enough-performance-now">available inChrome</a> as the <code>window.performance.now()</code> method.</p><h2 id="network-panel-overview">Network panel overview</h2><p>The Network panel automatically records all network activity while DevTools is open. The first time you open the panel it may be empty. Reload the page to start recording, or simply wait for network activity to occur in your application.</p><p><img src="network-files/panel.png" alt="alt text"/></p><p>Each requested resource is added as a row to the Network table, which contains the columns listed below. Note the following about the Network table:</p><ul><li>Not all columns listed below are visible by default; you can easily <a href="network.md#adding-and-removing-table-columns">show or hide columns</a>.</li><li>Some columns contain a primary field and a secondary field (<strong>Time</strong> and <strong>Latency</strong>, for example). When viewing the Network table with <a href="network.md#changing-resource-row-sizes">large resource rows</a> both fields are shown; when using small resource rows only the primary field is shown.</li><li>You can <a href="network.md#sorting-and-filtering">sort</a> the table by a column's value by clicking the column header. The <a href="network.md#timeline-view">the Timeline column</a> behaves a bit differently: clicking its column header displays a menu of additional sort fields. See <a href="network.md#timeline-view">Timeline view</a> and <a href="network.md#sorting-and-filtering">Sorting and filtering</a> for more information.</li></ul><table><thead><tr><th width="20%">Field</th><th>Description</th></tr></thead><tbody><tr><td><strong>Name</strong> and <strong>Path</strong></td><td>The name and URL path of the resource, respectively.</td></tr><tr><td><strong>Method</strong></td><td>The HTTP method used for the request (GET or POST, for example).</td></tr><tr><td><strong>Status</strong> and <strong>Text</strong></td><td>The HTTP status code and text message, respectively.</td></tr><tr><td><strong>Domain</strong></td><td>The domain of the resource request.</td></tr><tr><td><strong>Type</strong></td><td>The MIME type of the requested resource.</td></tr><tr><td><strong>Initiator</strong></td><td>The object or process that initiated the request. It can have one of the following values:  <dl>    <dt>Parser</dt>    <dd>Chrome's HTML parser initiated the request.</dd>    <dt>Redirect</dt>    <dd>A HTTP redirect initiated the request.</dd>    <dt>Script</dt>    <dd>A script initiated the request.</dd>    <dt>Other</dt>    <dd>Some other process or action initiated the request, such as the user navigating to a page via a link, or by entering a URL in the address bar.</dd>  </dl></td></tr><tr><td><strong>Cookies</strong></td><td>The number of cookies transferred in the request. These correspond to the cookies shown in the <a href="network.md#cookies">Cookies tab</a> when viewing details for a given resource.</td></tr><tr><td><strong>Set-Cookies</strong></td><td>The number of cookies set in the HTTP request.</td></tr><tr><td><strong>Size</strong> and <strong>Content</strong></td><td>Size is the combined size of the response headers (usually a few hundred bytes) plus the response body, as delivered by the server.Content is the size of the resource's decoded content.If the resource was loaded from the browser's cache rather than over the network, this field will contain the text (from cache).</td></tr><tr><td><strong>Time</strong> and <strong>Latency</strong></td><td>Time is total duration, from the start of the request to the receipt of the final byte in the response.Latency is the time to load the first byte in the response.</td></tr><tr><td><strong>Timeline</strong></td><td>The Timeline column displays a <a href="network.md#timeline-view">timeline view</a> of all network requests. Clicking the header of this column reveals a menu of additional sorting fields. See Timeline view and Sorting and filtering for details.</td></tr></tbody></table><h3 id="preserving-the-network-log-upon-navigation">Preserving the network log upon navigation</h3><p>By default, the current network record log is discarded when you navigate to another page, or reload the current page. To preserve the recording log in these scenarios, click the black <strong>Preserve log upon navigation</strong> button <img src="network-files/keep-log-off.png" alt="alt text"/> at the bottom of the Network panel; new records are appended to the bottom of the table. Click the same button again (now red <img src="http://developer.chrome.com/devtools/docs/network-files/keep-log-on.png" alt="alt text"/> <!-- TODO: Fix alt text and URL -->) to disable log preservation.</p><h3 id="sorting-and-filtering">Sorting and filtering</h3><p>By default, resources in the Network table are sorted by the start time of each request (the network "waterfall"). You can sort the table by another column value by clicking the column header. Click the header again to change the sort order (ascending or descending).</p><p><img src="network-files/sorting.png" alt="alt text"/></p><p>The Timeline column is unique from the others in that, when clicked, it displays a menu of additional sort fields.</p><p><img src="network-files/timeline-column.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>The menu contains the following sorting options:</p><ul><li><strong>Timeline</strong> — Sorts by the start time of each network request. This is the default sort, and is the same as sorting by the Start Time option).</li><li><strong>Start Time</strong> — Sorts by the start time of each network request (same as sorting by the Timeline option).</li><li><strong>Response Time</strong> — Sorts by each request's response time.</li><li><strong>End Time</strong> — Sorts by the time when each request completed.</li><li><strong>Duration</strong> — Sorts by the total time of each request.</li><li><strong>Latency</strong> — Sorts by the time between the start of the request and the beginning of the response (also known as the "time to first byte").</li></ul><p>To filter the Network table to only show certain types of resources, click one of the content types along the bottom of the panel: <strong>Documents</strong>, <strong>Stylesheets</strong>, <strong>Images</strong>, <strong>Scripts</strong>, <strong>XHR</strong>, <strong>Fonts</strong>, <strong>WebSockets</strong>, and <strong>Other</strong>. To show or hide a column, Right+click or Control+click (Mac only) in the table header and select or deselect column names from the list.</p><p><img src="network-files/add-remove-columns.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><h3 id="changing-resource-row-sizes">Changing resource row sizes</h3><p>You can view the Network table with large resource rows (the default), or small resource rows. Click the blue <strong>Use small resource rows</strong> toggle button <img src="network-files/small-resource-rows.png" alt="alt text"/> at the bottom of the panel to view small rows. Click the same button (now gray <img src="network-files/large-rows.png" alt="alt text"/>) to view large resource rows again. Large rows enable some columns to display two text fields: a primary field and a secondary field (Time and Latency, for instance). When viewing small rows only the primary field is displayed.</p><p>In the following screenshot, the Network table is viewed with small resource rows and just the Timeline column.</p><p><img src="network-files/small-rows.png" alt=""></p><h3 id="timeline-view">Timeline view</h3><p>The Timeline view in the Network panel graphs the time it took to load each resource, from the start of the HTTP request to the receipt of the final byte of the response. Each resource loading time is represented as a bar, color-coded according to the resource type. The length of the lighter-shaded part of each bar represents the request's latency, while the length of the darker-shaded part represents the time spent receiving the response data.</p><p><img src="network-files/network-timeline.png" alt="Network timeline view"></p><p>When you hover your mouse over a timeline row (but not over an actual bar) the request's latency and receipt time are displayed above the corresponding bar's light- and dark-shaded areas, respectively, as shown below.</p><p><img src="network-files/timeline-view-1.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>If you hover your mouse over the timeline bar itself, the complete timing data for the resource is presented in a pop-up. This is the same information that's presented in the <a href="network.md#resource-network-timing">Timing details tab</a> for a given resource.</p><p><img src="network-files/timeline-view-hover.png" alt="alt text"/></p><p>The timeline indicates when the the <a href="http://docs.webplatform.org/wiki/dom/events/DOMContentLoaded"><code>DOMContentLoaded</code></a>and <a href="http://docs.webplatform.org/wiki/dom/events/load"><code>load</code></a> events were fired with blue and red vertical lines, respectively. The <code>DOMContentLoaded</code> event is fired when the main document had been loaded and parsed. The <code>load</code> event is fired when all of the page's resources have been downloaded.</p><p><img src="network-files/dom-lines.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>Timeline bars are color-coded as follows:</p><style>#colortable {  width: 50%;  border: none;}#colortable td {  border: none;}.doc { background: rgba(47, 102, 236, 0.6); width: 10%;}.css { background: rgba(157, 231, 119, 0.6);width: 10%;}.images { background: rgba(164, 60, 255, 0.6);width: 10%;}.scripts { background: rgba(255, 121, 0, 0.6);width: 10%;}.xhr { background: rgba(231, 231, 10, 0.6);width: 10%;}.fonts { background: rgba(255, 82, 62,0.6);width: 10%;}.other { background: rgba(187, 187, 188, 0.6);width: 10%;}</style><!-- TODO: Fix formatting of cells --><table id="colortable"><tr><td class="doc"></td><td>Documents</td></tr><tr><td class="css"></td><td>Stylesheets</td></tr><tr><td class="images"></td><td>Images</td></tr><tr><td class="scripts"></td><td>Scripts</td></tr><tr><td class="xhr"></td><td>XHR</td></tr><tr><td class="fonts"></td><td>Fonts</td></tr><tr><td class="other"></td><td>Other</td></tr></table><h3 id="saving-and-copying-network-information">Saving and copying network information</h3><p>Right+clicking or Control+clicking (Mac only) within the Network table a context menu appears with several actions. Some of these actions apply to the resource row under the mouse click (like <a href="network.md#copying-requests-as-curl-commands">copying HTTP request headers</a>), while others apply to the entire network recording (such as <a href="network.md#saving-network-data">saving a Network recording as a HAR file</a>).</p><p><img src="network-files/right-click.png" alt="alt text"/> <!-- TODO: Fix alt text and URL --></p><p>The following menu actions apply to the selected resource:</p><ul><li><strong>Open Link in New Tab</strong> — Opens the resource in a new tab.
