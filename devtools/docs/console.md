</header>    <main id="gc-pagecontent" role="main"><article class="article-content">  <div itemprop="articleBody"></p><h1 id="using-the-console">Using the Console</h1><p>The JavaScript Console provides two primary functions for developers  testing web pages and applications:</p><ul><li>A place to log diagnostic information using methods provided by the <a href="console-api.md">Console API</a>, such as <a href="console-api.md#consolelogobject-object">console.log()</a>, or <a href="console-api.md#consoleprofilelabel">console.profile()</a>.</li><li>A shell prompt where you can enter commands and interact with the document and the Chrome DevTools. You can evaluate expressions directly in the Console, and can also use the methods provided by the <a href="commandline-api.md">Command Line API</a>, such as <a href="commandline-api.md#selector">$()</a> command for selecting elements, or <a href="commandline-api.md#profilename">profile()</a> to start the CPU profiler.</li></ul><p>This documentation provides an overview and common uses of these two APIs. You can also browse the <a href="console-api.md">Console API</a> and <a href="commandline-api.md">Command Line API</a> reference guides.</p><h2 id="basic-operation">Basic operation</h2><h3 id="opening-the-console">Opening the Console</h3><p>The JavaScript Console is available in two modes within Chrome DevTools: the primary Console tab, or as a split-view you can display while on another tab (such as Elements or Sources).</p><p>To open the Console tab, do one of the following:</p><ul><li>Use the keyboard shortcut <strong>Command - Option - J</strong> (Mac) or <strong>Control -Shift -J</strong> (Windows/Linux).</li><li>Select <strong>View &gt; Developer &gt; JavaScript Console</strong>.</li></ul><p><img alt="Console panel view" src="console-files/console1.png" /></p><p>To toggle a split-view of the Console on another tab, press the <strong>Esc</strong> key on your keyboard, or click the <strong>Show/Hide Console</strong> button in the bottom left corner of the Chrome DevTools window. In the following screenshot the Console split-view is shown with the Elements panel.</p><p><img alt="Console split-view" src="console-files/console-split-view.png" /></p><h3 id="clearing-the-console-history">Clearing the console history</h3><p>To clear the console's history, do one of the following:</p><ul><li>Right-click or Ctrl-click anywhere in the Console and choose <strong>Clear Console</strong> from the context menu that appears.</li><li>Enter the <a href="commandline-api.md#clear"><strong>clear()</strong></a> Command Line API at the shell prompt.</li><li>Invoke <a href="console-api.md#consoleclear"><strong>console.clear()</strong></a> Console API from JavaScript.</li><li>Use the keyboard shortcut <strong>⌘K</strong> or <strong>⌃L</strong> (Mac) <strong>Control - L</strong> (Windows and Linux).</li></ul><p>By default, the console history is cleared when you navigate to another page. You can change this behavior by enabling <strong>Preserve log upon navigation</strong> in the Console area of the Settings dialog (see <a href="console.md#consolepreferences">Console preferences</a>).</p><h3 id="console-settings">Console settings</h3><p>The Console has two global settings you can modify in the General tab of the DevTools Settings dialog:</p><ul><li><strong>Log XMLHTTPRequests</strong>&mdash;determines if each XMLHTTPRequest is logged to the Console panel.</li><li><strong>Preserve log upon navigation</strong>&mdash;determines if console history for the current page is preserved when you navigate to another page. By default, both of these settings are disabled.</li></ul><p>You can also change these settings by right-clicking anywhere in the Console to bring up the context menu.</p><p><img alt="Console panel view" src="console-files/console-context-menu.png" /></p><h2 id="using-the-console-api">Using the Console API</h2><p>The Console API is collection of methods provided by the global <code>console</code> object defined by DevTools. One of the API's main purposes is to <a href="console.md#writing-to-the-console">log information</a> (such as a property value, or an entire objects or DOM element) to the console while your application is running. You can also group output visually in the console to reduce visual clutter.</p><h3 id="writing-to-the-console">Writing to the console</h3><p>The <a href="console-api.md#consolelogobject-object">console.log()</a> method takes one or more expressions as parameters and writes their current values to the console. For example:</p><pre><code>var a = document.createElement('p');a.appendChild(document.createTextNode('foo'));a.appendChild(document.createTextNode('bar'));console.log("Node count: " + a.childNodes.length);</code></pre><p><img alt="Console log output" src="console-files/log-basic.png" /></p><p>Instead of concatenating expressions together with the "+" operator (as shown above), you can put each in its own method parameter and they will be joined together in a space-delimited line.</p><pre><code>console.log("Node count:", a.childNodes.length, "and the current time is:", Date.now());</code></pre><p><img alt="Console log output" src="console-files/log-multiple.png" /></p><h3 id="errors-and-warnings">Errors and warnings</h3><p>The <a href="console-api.md#consoleerrorobject-object"><code>console.error()</code></a> method displays a red icon along with the message text, which is colored red.</p><pre><code>function connectToServer() {    console.error("Error: %s (%i)", "Server is  not responding",500);}connectToServer();</code></pre><p><img alt="" src="console-files/error-server-not-resp.png" /></p><p>The <a href="console-api.md#consolewarnobject-object"><code>console.warn()</code></a> method displays a yellow warning icon with the message text.</p><pre><code>if(a.childNodes.length &lt; 3 ) {    console.warn('Warning! Too few nodes (%d)', a.childNodes.length);}</code></pre><p><img alt="Example of console.warn()" src="console-files/warning-too-few-nodes.png" /></p><h3 id="assertions">Assertions</h3><p>The <a href="console-api.md#consoleassertexpression-object"><code>console.assert()</code></a> method conditionally displays an error string (its second parameter) only if its first parameter evaluates to <code>false</code>. For instance, in the following example an error message is written to the console only if the number of child nodes belonging to the <code>list</code> element is greater than 500.</p><pre><code>console.assert(list.childNodes.length &lt; 500, "Node count is &gt; 500");</code></pre><p><img alt="Example of console.assert()" src="console-files/assert-failed.png" /></p><h3 id="filtering-console-output">Filtering console output</h3><p>You can quickly filter console output by its severity level--errors, warning, or standard log statements--by selecting one of the filter options. Click on the filter funnel (as shown below) and then select which filter you want to use.</p><p><img alt="Only show console.error() output" src="console-files/filter-errors.png" /></p><p>Filter options:</p><ul><li><strong>All</strong>&mdash;Shows all console output.</li><li><strong>Errors</strong>&mdash;Only show output from <code>console.error()</code></li><li><strong>Warnings</strong>&mdash;Only show output from <code>console.warn()</code></li><li><strong>Logs</strong>&mdash;Only show output from <code>console.log()</code>, <code>console.info()</code> and <code>console.debug()</code>.</li><li><strong>Debug</strong>&mdash;Only show output from <code>console.timeEnd()</code> and other console output.</li></ul><h3 id="grouping-output">Grouping output</h3><p>You can visually group related console output statements together in the console with the <a href="console-api.md#consolegroupobject-object"><code>console.group()</code></a> and <a href="console-api.md#consolegroupend"><code>groupEnd()</code></a> commands.</p><pre class ="prettyprint">    var user = "jsmith", authenticated = false;    console.group("Authentication phase");    console.log("Authenticating user '%s'", user);    // authentication code here...    if (!authenticated) {        console.log("User '%s' not authenticated.", user)    }    console.groupEnd();</pre><p><img alt="Logging group example" src="console-files/group.png" /></p><p>You can also nest logging groups. In the following example a logging group is created for the authentication phase of a login process. If the user is authenticated, a nested group is created for the authorization phase.</p><pre><code>var user = "jsmith", authenticated = true, authorized = true;// Top-level groupconsole.group("Authenticating user '%s'", user);if (authenticated) {    console.log("User '%s' was authenticated", user);    // Start nested group    console.group("Authorizing user '%s'", user);    if (authorized) {        console.log("User '%s' was authorized.", user);    }    // End nested group    console.groupEnd();}// End top-level groupconsole.groupEnd();console.log("A group-less log trace.");</code></pre><p><img alt="Nested logging group example" src="console-files/nestedgroup.png" /></p><p>To create a group that is initially collapsed, use <a href="console-api.md#consolegroupcollapsed"><code>console.groupCollapsed()</code></a> instead of <code>console.group()</code>, as shown below:</p><pre><code>console.groupCollapsed("Authenticating user '%s'", user);if (authenticated) {  ...}</code></pre><p><img alt="Initially collapsed group" src="console-files/groupcollapsed.png" /></p><h3 id="string-substitution-and-formatting">String substitution and formatting</h3><p>The first parameter you pass to any of the console's logging methods (<code>log()</code> or <code>error()</code>, for example) may contain one or more <em>format specifiers</em>. A format specifier consists of a <strong><code>%</code></strong> symbol followed by a letter that indicates the formatting that should be applied to the inserted value (<code>%s</code> for strings, for example). The format specifier identifies where to substitute a value provided by a subsequent parameter value.</p><p>The following example using the <code>%s</code> (string) and <code>%d</code> (integer) formatters to insert values into the output string.</p><pre><code>console.log("%s has %d points", "Sam", "100");</code></pre><p>This would result in "Sam has 100 points" being logged to the console.</p><p>The following table lists the supported format specifiers and the formatting they apply:</p><table><thead><tr><th>Format specifier</th><th>Description</th></tr></thead><tbody><tr><td><code>%s</code></td><td>Formats the value as a string.</td></tr><tr><td><code>%d</code> or <code>%i</code></td><td>Formats the value as an integer.</td></tr><tr><td><code>%f</code></td><td>Formats the object as a floating point value.</td></tr><tr><td><code>%o</code></td><td>Formats the value as an expandable DOM element (as in the Elements panel).</td></tr><tr><td><code>%O</code></td><td>Formats the value as an expandable JavaScript object.</td></tr><tr><td><code>%c</code></td><td>Applies CSS style rules to output string specified by the second parameter.</td></tr></tbody></table><p>In the following example the <code>%d</code> format specifier is substituted with the value of <code>document.childNodes.length</code> and formatted as an integer; the <code>%f</code> format specifier is substituted with the value returned by <code>Date.now()</code>, which is formatted as a floating point number.</p><pre><code>console.log("Node count: %d, and the time is %f.", document.childNodes.length, Date.now());</code></pre><p><img alt="Using format specifiers" src="console-files/format-substitution.png" /></p><h3 id="formatting-dom-elements-as-javascript-objects">Formatting DOM elements as JavaScript objects</h3><p>By default, when you log a DOM element to the console it's displayed in an XML format, as in the Elements panel:</p><pre><code>console.log(document.body.firstElementChild)</code></pre><p><img alt="" src="console-files/log-element.png" /></p><p>You can also log an element's JavaScript representation with the <code>console.dir()</code> method:</p><pre><code>console.dir(document.body.firstElementChild);</code></pre><p><img alt="" src="console-files/dir-element.png" /></p><p>Equivalently, you can us the <code>%O</code> <a href="console.md#string-substitution-and-formatting">format specifier</a> with <code>console.log()</code>:</p><pre><code>console.log("%O", document.body.firstElementChild);</code></pre><h3 id="styling-console-output-with-css">Styling console output with CSS</h3><p>You use the <code>%c</code> format specifier to apply custom CSS rules to any string you write to the Console with <a href="console.md#writingtotheconsole"><code>console.log()</code></a> or related methods.</p><pre><code>console.log("%cThis will be formatted with large, blue text", "color: blue; font-size: x-large");</code></pre><p><img alt="Styling console output with CSS" src="console-files/format-string.png" /></p><h3 id="measuring-how-long-something-takes">Measuring how long something takes</h3><p>You can use the <a href="console-api.md#consoletimelabel"><code>console.time()</code></a> and <a href="console-api.md#consoletimeendlabel"><code>console.timeEnd()</code></a> methods to measure how long a function or operation in your code takes to complete. You call <code>console.time()</code> at the point in your code where you want to start the timer and <code>console.timeEnd()</code> to stop the timer. The elapsed time between these two calls is displayed in the console.</p><pre><code>console.time("Array initialize");var array= new Array(1000000);for (var i = array.length - 1; i &gt;= 0; i--) {    array[i] = new Object();};console.timeEnd("Array initialize");</code></pre><p><img alt="Example of using console.time() and timeEnd()" src="console-files/time-duration.png" /></p><p>Note: You must pass the same string to <code>console.time()</code> and <code>timeEnd()</code> for the timer to finish as expected.</p><h3 id="marking-the-timeline">Marking the Timeline</h3><p>The <a href="timeline.md">Timeline panel</a> gives you a complete overview of where time is spent when loading and using your web app or page. The <a href="console-api.md#consoletimestamplabel"><code>console.timeStamp()</code></a> method marks the Timeline at the moment it was executed. This provides an easy way to correlate events in your application with other browser-related events, such as layout or paints.</p><p>Note: The <code>console.timeStamp()</code> method only functions while a Timeline recording is in progress.</p><p>In the following example the Timeline is marked when the application enters the <code>AddResult()</code> function's implementation.</p><pre><code>function AddResult(name, result) {  console.timeStamp("Adding result");  var text = name + ': ' + result;  var results = document.getElementById("results");  results.innerHTML += (text + "&lt;br&gt;");}</code></pre><p>As shown in the following screenshot, the <code>timeStamp()</code> command annotates the Timeline in the following places:</p><ul><li>A yellow vertical line in the Timeline's summary and detail views.</li><li>A record is added to the list of recorded events.</li></ul><p><img alt="Timeline showing custom timestamp" src="console-files/timestamp2.png" /></p><h3 id="setting-breakpoints-in-javascript">Setting breakpoints in JavaScript</h3><p>You can start a debugging session from your JavaScript code by calling the <a href="console-api.md#debugger"><code>debugger</code></a> command. For instance, in the following example the JavaScript debugger is opened when an object's <code>brightness()</code> function is invoked:</p><pre><code>brightness : function() {    debugger;    var r = Math.floor(this.red*255);    var g = Math.floor(this.green*255);    var b = Math.floor(this.blue*255);    return (r * 77 + g * 150 + b * 29) &gt;&gt; 8;}</code></pre><p><img alt="Example of using debugger command" src="console-files/debugger.png" /></p><aside class="special">An interesting technique of using conditional breakpoints was explored by Brian Arnold in <a href="http://www.randomthink.net/blog/2012/11/breakpoint-actions-in-javascript/">Breakpoint Actions in JavaScript</a>.</aside><h2 id="using-the-command-line-api">Using the Command Line API</h2><p>In addition to being a place where you can log information from your application, the Console is also a shell prompt where you can directly evaluate expressions or issue commands provided by the <a href="commandline-api.md">Command Line API</a>. This API provides the following features:</p><ul><li>Convenience functions for selecting DOM elements</li><li>Methods for controlling the CPU profiler</li><li>Aliases for a number of Console API methods</li><li>Monitoring events</li><li>View event listeners registered on objects</li></ul><h3 id="evaluating-expressions">Evaluating expressions</h3><p>The Console attempts to evaluate any JavaScript expression you enter at the shell prompt, upon pressing the Return or Enter key. The Console provides auto-completion and tab-completion. As you type expressions, property names are automatically suggested. If there are multiple properties with the same prefix, pressing the Tab key cycles through them. Pressing the right arrow key accepts the current suggestion. The current suggestion is also accepted by pressing the Tab key if there is only one matched property.</p><p><img alt="" src="console-files/evaluate-expressions.png" /></p><p>To enter a multi-line expression at the shell prompt (such as a function definition) press Shift+Enter between lines.</p><p><img alt="" src="console-files/multiline-expression.png" /></p><h3 id="selecting-elements">Selecting elements</h3><p>The Command Line API provides several methods to access DOM elements in your application. For example, the <a href="commandline-api.md#selector"><strong><code>$()</code></strong></a> method returns the first element that matches the specified CSS selector, just like <a href="http://docs.webplatform.org/wiki/css/selectors_api/querySelector"><code>document.querySelector()</code></a>. For instance, the following code returns the element with the ID "loginBtn".</p><pre><code>$('#loginBtn');</code></pre><p><img alt="" src="console-files/select-login-btn.png" /></p><p>The <a href="commandline-api.md#selector-1"><strong><code>$$()</code></strong></a> command returns an array of all the elements that match the specified CSS selector, just like <a href="http://docs.webplatform.org/wiki/css/selectors_api/querySelectorAll"><code>document.querySelectorAll()</code></a>. For instance, the following displays selects all <code>&lt;button&gt;</code> elements with the CSS class "loginBtn":</p><pre><code>$$('button.loginBtn');</code></pre><p><img alt="" src="console-files/select-multiple-login.png" /></p><p>Lastly, the <a href="commandline-api.md#xpath"><code>x()</code></a> method takes an XPath path as a parameter and returns an array of all elements that match the specified path. The following returns all the &lt;script&gt; elements that are children of the <code>&lt;body&gt;</code> tag:</p><pre><code>$x('/html/body/script');</code></pre><h3 id="inspecting-dom-elements-and-javascript-heap-objects">Inspecting DOM elements and JavaScript heap objects</h3><p>The <a href="commandline-api.md#inspectobject"><code>inspect()</code></a> method takes a DOM element reference (or JavaScript reference) as a parameter and displays the element or object in the appropriate panel&mdash;the Elements panel for DOM elements, or the Profile panel for a JavaScript object.</p><p>For example, in the following screenshot the <code>$()</code> function is used to get a reference to an <code>&lt;li&gt;</code> element. Then the last evaluated expression property (<a href="commandline-api.md#_"><code>$_</code></a>) is passed to <code>inspect()</code> to open that element in the Elements panel.</p><p><img alt="" src="console-files/inspect2.png" /></p><h3 id="accessing-recently-selected-elements-and-objects">Accessing recently selected elements and objects</h3><p>Often when testing you'll select DOM elements&mdash;either directly in the Elements panel or using the Selection tool (magnifying glass)&mdash;so that you can further inspect the element. Or, when analyzing a memory snapshot in the Profiles panel, you might select a JavaScript object to further inspect it.</p><p>The Console remembers the last five elements (or heap objects) you've selected and makes them available as properties named <a href="commandline-api.md#0-4"><strong><code>$0</code></strong>, <strong><code>$1</code></strong>, <strong><code>$2</code></strong>, <strong><code>$3</code></strong></a> and <a href="commandline-api.md#0-4"><strong><code>$4</code></strong></a>. The most recently selected element or object is available as <strong><code>$0</code></strong>, the second most as <strong><code>$1</code></strong>, and so forth.</p><p>The following screenshot shows the values of these properties after selecting three different elements in turn from the Elements panel:</p><p><img alt="Recently selected elements" src="console-files/recent-selection.png" /></p><p>Note: You can also Right-click or Control-click on any element in the Console and select <strong>Reveal in Elements Panel</strong>.</p><h3 id="monitoring-events">Monitoring events</h3><p>The <a href="commandline-api.md#monitoreventsobject-events"><code>monitorEvents()</code></a> command monitors an object for one or more specified events. When an event occurs on the monitored object, the corresponding Event object is logged to the Console. You specify the object and the events you want to monitor on that object. For example, the following code enables event monitoring for every "resize" event on the global window object.</p><pre><code>monitorEvents(window, "resize");</code></pre><p><img alt="Monitoring window resize events" src="console-files/monitor-resize.png" /></p><p>To monitor several events, you can pass an array of event names as the second parameter. The code below monitors both "mousedown" and "mouseup" events on the body of the document.</p><pre><code>monitorEvents(document.body, ["mousedown", "mouseup"]);</code></pre><p>You can also pass one of the supported "event types" that DevTools maps to a set of actual event names. For example, the "touch" event type cause DevTools to monitor "touchstart", "touchend", "touchmove", and "touchcancel" events on the target object.</p><pre><code>monitorEvents($('#scrollBar'), "touch");</code></pre><p>See <a href="commandline-api.md#monitoreventsobject-events"><code>monitorEvents()</code></a> in the Console API Reference for a list of supported event types.</p><p>To stop monitoring events call <code>unmonitorEvents()</code>, passing the object to stop monitoring.</p><pre><code>unmonitorEvents(window);</code></pre><h3 id="controlling-the-cpu-profiler">Controlling the CPU profiler</h3><p>You can create JavaScript CPU profiles from the command line with the <a href="commandline-api.md#profilename"><code>profile()</code></a> and <a href="commandline-api.md#profileendname"><code>profileEnd()</code></a> commands. You can optionally specify a name that's applied to the profile you create.</p><p>For example, the following shows an example of creating a new profile with the default name:</p><p><img alt="" src="commandline-api-files/profile-console.png" /></p><p>The new profile appears in the Profiles panel under the name "Profile 1":</p><p><img alt="" src="commandline-api-files/profile-panel.png" /></p><p>If you specify a label for the new profile, it is used as the new profile's heading. If you create multiple profiles with the same name, they are grouped as individual runs under the same heading:</p><p><img alt="" src="commandline-api-files/profile-console-2.png" /></p><p>The result in the Profiles panel:</p><p><img alt="" src="commandline-api-files/profile-panel-2.png" /></p><p>CPU profiles can be nested, for example:</p><pre><code>profile("A");profile("B");profileEnd("B")profileEnd("A")</code></pre><p>The calls to stop and start profiling do not need be properly nested. For example, the following works the same as the previous example:</p><pre><code>profile("A");profile("B");profileEnd("A");profileEnd("B");</code></pre><p>        <footer id="cc-info">      <button id="send-feedback" class="google-button" data-feedback>Send Feedback</button>      <!--p class="last-updated">Last updated August 2, 2013.</p-->      <p class="cc-logo"><a href="http://creativecommons.org/licenses/by/3.0/"><img src="../../static/images/cc_by.png" alt="Licensed Creative Commons by Attribution"></a></p>    </footer>    <nav class="inline-toc no-permalink">      <section class="related">                  <h3 title="">Learn Basics</h3>      <ol class="toc">                        <li>                          <a href="../index.html">Overview</a>                        </li>                        <li>                          <a href="authoring-development-workflow.html">Authoring &amp; Development Workflow</a>                        </li>                        <li>                          <a href="console.md" class="active">Using the Console</a>                        </li>                        <li>                          <a href="tips-and-tricks.html">Tips &amp; Tricks</a>                        </li>                        <li>                          <a href="videos.html">Additional Resources</a>                        </li>      </ol>      </section>      <section id="toc">
